Problem Statement

Traditional library management was done manually — tracking which books were issued, calculating fines, and managing members required paperwork and was prone to human error. The goal was to build a simple, efficient digital system to automate these processes.

What I Built
  • Book issue and return system with due date tracking
  • Automatic fine calculation based on return delay
  • Member registration and data management
  • Book catalog with search and availability status
  • File-based persistent storage using Java I/O
  • Clean GUI built with Java Swing components
Technical Decisions

I used Java Collections (ArrayList, HashMap) for in-memory data management and File I/O for persistent storage, keeping the system lightweight with no database dependency. Applied core OOP principles — encapsulation, inheritance, and abstraction — to keep the codebase modular and maintainable.

Future Enhancements
  • Database integration (MySQL/SQLite) to replace file-based storage
  • Real-time data handling and multi-user support
  • Web-based interface using React.js frontend
  • Email/SMS notification for due dates
